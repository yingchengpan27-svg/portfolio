## 1. 项目定位与关键决策
- **定位**：面向 PC 与移动端的单页作品集网站，身份定义为“新媒体全栈操盘手 / AI 视觉创作者”。
- **整体风格**：暗色系 + 科技感 + 克制高级感，避免模板化，重点突出视觉氛围与内容层级。
- **技术选型**：
  - 使用 **React 19 + Vite 8** 构建。
  - 页面结构采用纯前端单页方式，无路由、无后端业务逻辑。
  - 以组件化方式拆分页面模块：Navbar、Hero、About、Projects、Skills、Contact。
- **布局策略**：
  - 版心控制在 `1700px`。
  - Hero 模块保留“参考1.png”素材作为头图人物背景。
  - Hero 当前已按需求把背景人物整体向右调整，避免遮挡左侧文案。
- **视觉系统**：
  - 定义在 `src/styles/global.css` 的设计变量中。
  - 强调紫色/靛蓝渐变、柔和发光、细边框、暗色背景层次。

## 6. 整体架构思路
### 6.1 结构层
- 单页结构：一个主页依次承载 Hero、About、Projects、Skills、Contact。
- 组件拆分：每个页面模块独立组件，便于后续分别优化。

### 6.2 样式层
- 全局设计变量集中在 `src/styles/global.css`。
- 各组件使用 `<style>` 块写局部样式，当前项目属于轻量快速实现阶段。
- 版心统一控制在 `1700px`，通过 `.container` 约束。

### 6.3 交互层
- Navbar 滚动变化。
- Hero 背景粒子动画。
- 按钮 hover / 卡片 hover 动效。
- 后续适合继续增强，但当前先保证视觉准确。

### 6.4 内容层
- 网站文案已经来源于你的简历信息。
- 项目卡片目前为文案 + 占位视觉，下一步重点是替换真实作品素材。
- Hero 已经从纯抽象背景进化为“参考图 + 文案”的更具个人特征版本。
